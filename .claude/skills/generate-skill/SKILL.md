# スキル作成ガイド

このスキルは、Claude Codeが「意図通りに動く」ための高品質なスキルを作成するための指針です。 **「スキルの作り方をスキルで定義する」** ことにより、開発プロセスの一貫性を保ちます。

## 1. スキルの基本構造

スキルは単一のファイルではなく、**ディレクトリ構造**で管理し、情報を「段階的開示（Progressive Disclosure）」できるように構成してください,。

- **配置場所**:
    - プロジェクト固有: `.claude/skills/[スキル名]/SKILL.md`
    - ユーザー全体（汎用）: `~/.claude/skills/[スキル名]/SKILL.md`
- **ディレクトリ構成の例**:
    - `SKILL.md`: スキルの本体（必須）
    - `examples/`: 良いコード例、悪いコード例
    - `templates/`: 雛形となるファイル
    - `best-practices.md`: 詳細なガイドライン

## 2. フロントマター（メタデータ）の定義ルール

Claude Codeが「いつこのスキルをロードすべきか」を判断するための重要なセクションです,。

- **name**: スキルの一意な識別子（小文字とハイフンを使用）。
- **description (最重要)**: 具体的な**トリガーワード**を含めて記述してください,。
    - **良い例**: 「Reactコンポーネント作成時、Hooks使用時に使用。コンポーネントの構成、状態管理、イベントハンドリングのパターンを定義」。
    - **悪い例**: 「Reactのヘルプ」。
- **allowed-tools**: 必要に応じて、そのスキル内で使用を許可するツールを制限します。

## 3. 効果的なスキルを作成するための3原則

指示追従性を高め、推論のパフォーマンスを最大3倍向上させるための原則です,,。

### ① ツールを明示的に指定する
「どのツールを使って何をするか」を具体的に記述してください。
- **例**: 「新しい機能を実装する前に、 **Grep ツール** で既存パターンを検索してください」。

### ② 実行順序を具体的に記述する
ツールをどの順番で使うかをステップバイステップで示してください。
- **例**:
    1. **Grep** で既存パターンを検索
    2. **Read** で既存実装を確認
    3. パターンに従って実装
    4. **Bash** でテストを実行

### ③ 具体的なコード例を含める
抽象的なルールだけでなく、期待される動作をコード例（推奨パターン）で示してください,。
- **理由**: 自然言語による説明よりも、構造化された技術要素（テストコードやメソッド名）を提示したほうが成功率が高まることが研究で示唆されています。

## 4. CLAUDE.md やスラッシュコマンドとの使い分け

- **CLAUDE.md**: プロジェクト全体で「常に知っておくべきこと」を記述（常時コンテキストを消費）。
- **スラッシュコマンド**: 頻繁に実行する「ワークフロー（何をするか、どの順番でするか）」を定義。
- **スキル**: 特定の状況で必要な「考え方やツールの使い方」を定義。必要な時だけロードされるため、**コンテキストの節約**に直結します,,。

## 5. ロードの仕組み（段階的開示）

スキルは以下の3段階で読み込まれます。この特性を理解し、情報を整理してください。
1. **第1段階**: メタデータ（name, description）をロードし、関連性を判断。
2. **第2段階**: `SKILL.md` 本文（目安5,000トークン未満）をロード。
3. **第3段階**: 必要に応じて補助ファイル群（examples等）をロード。
