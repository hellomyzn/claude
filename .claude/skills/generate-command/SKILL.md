## 1. このスキルの核心メッセージ
**「頻繁に実行する一連の手順（ワークフロー）をMarkdownで明文化し、変数や外部コマンドを組み合わせて『しくみ化』することで、AIの推論漏れを防ぎ、チーム全体での開発品質と速度を均一化する」**

---

## 2. 主要トピックの深掘り

### **トピックA：ワークフローの明示化と定型化**
*   **解決する課題**: 
    *   人間が手作業で行う際に含まれがちな「暗黙的な手順」の漏れ。
    *   人によって手順が異なる「属人化」の防止。
*   **仕組み・動作原理**: 
    *   Markdownファイル内に「何」を「どの順番で」実行すべきか具体的な手順を記述する。
*   **メリット・注意点**: 
    *   メリット：一度定義すれば、チーム全体で高品質なワークフローを共有できる。
    *   注意点：手順が曖昧だとAIが迷うため、できる限り具体的に記述する必要がある。

### **トピックB：YAMLフロントマターによるメタデータ設定**
*   **解決する課題**: 
    *   コマンドの用途制限や、実行時の動作（使用モデルやツール権限）の制御。
*   **仕組み・動作原理**: 
    *   ファイル冒頭にYAML形式で属性を定義する。
*   **各項目の役割**: 
    *   `description`: `/help` で表示される説明文。
    *   `allowed-tools`: そのコマンド実行中に許可するツールを制限し、セキュリティや精度を高める。
    *   `argument-hint`: ユーザーに表示する引数のヒント。
    *   `disable-model-invocation`: 推論を無効化し、純粋なスクリプト実行として扱う設定。

### **トピックC：動的要素の活用（変数・参照・コマンド）**
*   **解決する課題**: 
    *   固定的な指示だけでなく、実行時の状況に応じた柔軟な処理の実現。
*   **仕組み・動作原理**: 
    *   **変数 (`$ARGUMENTS`)**: 実行時に渡された引数をプロンプトに展開する。
    *   **ファイル参照 (`@`)**: 特定のドキュメントやスタイルガイドをコンテキストに強制注入する。
    *   **Bashコマンド実行 (`!`)**: `!git status` のように実行結果をそのままプロンプトの材料（コンテキスト）として利用する。

## 3. 実務で役立つ具体的なTips・仕様

### **配置場所と優先順位**
*   **プロジェクト固有**: `.claude/commands/` 配下。
    *   チーム開発のルールや特定プロジェクトの手順を定義する。
*   **ユーザー全体（汎用）**: `~/.claude/commands/` 配下。
    *   コードレビューのテンプレートなど、全プロジェクトで使いたい自分専用コマンドを定義する。
*   **優先順位**: 同名のコマンドがある場合、 **プロジェクトレベルが優先される** 。

### **効果的なコマンド作成の3原則**
1.  **単一責任の原則**: 
    *   1つのコマンドは1つの目的に集中させる（例：`/add-feature` は新機能追加のみ）。
2.  **適切な粒度**: 
    *   「1回の作業セッションで完了する一連の手順」を目安にする。
    *   大きすぎると混乱し、小さすぎるとReadツールの代わりになってしまう。
3.  **明確な実行フロー**: 
    *   「ステップ1：準備」「ステップ2：調査」のように、順序を番号付きリストで明記する。

## 4. 登場する専門用語ミニ辞書

*   **スラッシュコマンド**: 頻繁に使用するプロンプトをMarkdownファイルとして定義し、簡単に呼び出せるようにした機能。
*   **フロントマター**: Markdownファイルの先頭にある、コマンドの設定を記述したYAMLセクション。
*   **$ARGUMENTS**: ユーザーがコマンドの後に入力した文字列が格納される特別な変数。
*   **段階的開示**: スキルとは異なり、コマンドは実行された瞬間に定義された全手順がコンテキストに読み込まれる。
*   **コンテキストへの注入**: `@` や `!` を使って、外部情報やコマンド結果をAIの「記憶」に強制的に追加すること。
